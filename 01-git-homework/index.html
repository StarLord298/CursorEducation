01-git-homework

  • git config --global user.name blabla встановлюю глобально ім'я користувача
  • git config --global user.email blabal@bla встановлюю глобально електронну пошту
  • git init ініціалізація репозиторію, по замовчуванню створюється гілка master
  • git status показує стан файлів у робочих директоріях: які файли оновлено, але не додано до індексу; які очікують комміту в індексі.
  • git add додає вміст робочої директорії в індекс для комміту
  • git commit зберігає всі дані, що були добавлені в індекс
  • git log показує історію коммітів
  • git checkout копіює файли з історії або сцени в робочу директорію, можна переключатися між гілками
  • git branch створює нові, видаляє, перейменовує гілки, а також перераховує їх
  • git push вигружає вміст локального репозиторію в віддаленний
  • git pull викачує вміст віддаленного репозиторію
  • git merge злиття однієї або декількох гілок в одну
  • git clone клонує віддалений репозиторій
  • git revert Скасовує внесені в комміті зміни і додає новий комміт з отриманим вмістом.
  • git reset Скасовує зміни. Вона змінює вказівник HEAD і, опціонально, стан індексу. Також ця команда може змінити файли в робочій директорії при використанні параметра --hard
  • git rebase Переносить зміни з однієї гілки в іншу. Ця команда стискає всі зміни в один патч а тоді переносить зміни в цілеву гілку
  • git cherry-pick перенесення окремих коммітів з одного місця сховища в інше, зазвичай між гілками розробки та обслуговування.
  • git stash призначена для того, щоб помістити поточні зміни, які ви виконали в файлах, в окреме сховище, і повернути файли до вихідного стану

.gitignore служить для вказівки в ньому файлів і папок, які необхідно приховати від системи контролю версій git (робочі файли або файли налаштувань - як приклад node_modules)
